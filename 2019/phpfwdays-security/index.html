How to store passwords?

Hashing (even with salt) is a bad idea!

Popular hash functions

crc32, md5, sha128, sha256, sha512

are meant to be fast

Cryptographic hash functions

PBKDF2, bcrypt, scrypt

are meant to be slow

8-chars of [A-Za-z0-9] give 628 unique passwords [1]


8x Nvidia GTX 1080 GPU cards benchmark [2]:

  • md5 ‚Äì¬†200 GH/s, 18 minutes
  • sha256 ‚Äì¬†23012 MH/s, 2.5 hours
  • bcrypt¬†‚Äì¬†105 kH/s, 66 years
  • PBKDF2-HMAC-SHA512 ‚Äì¬†3450 kH/s, 3 years
  • Argon2 ‚Äì¬†GPU is useless
argon2 requires lots of memory access which spoils parallelism, complexity can be adjusted with iterations, a lot of free GPUs after mining

Strong hash function only gives you time to react
