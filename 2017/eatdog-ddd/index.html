DDD – ключевые концепции:

  • Единый язык домена (Ubiquitous Language)
  • Проектирование по модели (Model-Driven Design)
  • Шаблоны проектирования:
    • Entity
    • Value Object
    • Aggregation Root
    • Repository
    • Service

Единый язык домена

Набор терминов, применяемых в модели домена для объяснения связей и действий

Зачем?

  • общаться с доменными экспертами на одном языке
  • называть вещи своими именами
  • отслеживать изменения домена через изменения языка

Entity

Value Object

  • Определяется по значению аттрибутов, а не по идентификатору
